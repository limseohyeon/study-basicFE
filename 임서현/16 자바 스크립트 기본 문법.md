## 1.1 변수 종류

|  |  | 범위 | 특징 |
| --- | --- | --- | --- |
| `let` | 변수 | 해당 스코프 | 재선언 불가 |
| `var` | 변수 | 전역 스코프 | 재선언 가능 |
| `const` | 상수 | 해당 스코프 |  |
- **재선언** : 한 변수를 다시 한 번 선언해 값을 할당하는 것을 의미한다.
    
    ```jsx
    var v1 = 0;
    var v1 = 1;
    ```
    

## 1.2 var 변수 주의 *(*chapter17 내용 포함)*

var은 아래와 같은 이유로 사용을 지양하는 것이 좋다.

1. **전역 변수**
    
    자바스크립트에서는 지역 변수 선언 시 var 예약어를 빠뜨리면 해당 변수를 전역변수로 취급한다.
    
2. **호이스팅**
    
    코드 실행 전 자바 스크립트 해석기가 코드 평가 과정을 거치는데 , 이 때 ‘변수 선언문’을 미리 실행해두기 때문에 뒤에서 선언된 변수도 앞의 코드에서 참조할 수 있게 된다.
    
    아래는 호이스팅의 예시로 1번에서 a의 값을 로그로 확인하면 분명 a는 선언되지 않았는데 undefined 뜬다.  이는 var은 전역변수이기 때문에 코드를 실행하기 전이지만 기억해두고 선언한 것과 같은 효과가 난다. 
    
    ```jsx
    // (1)
    console.log(a); // undefined
    
    var a = 10;
    
    // (2)
    console.log(a); // 10
    ```
    
    호이스팅이 무조건 나쁜 것은 아니지만 프로그램 규모가 커질수록 이에 대한 문제가 발생할 가능성이 높고, 또한 이는 변수 뿐만 아니라 함수에도 적용되는 사항이기에 var의 사용은 지양하는 것이 좋다.
    
    - undefined : 변수는 선언 됐지만 값이 할당되지 않은 경우
3. **재선언**
    
    var의 특징인 재선언 자체가 예상치 못한 오류를 발생 시킬 수 있다.
    

## 1.3 자바스크립트 변수 사용하기

- 전역 변수는 최소한으로 사용한다.
- var 보다 let이나 const를 사용한다.
- for문에서 카운터 변수를 사용할 때는 let 예약어를 사용한다.
- 어떤 것을 상수로 어떤 것을 변수로 선언해야 할까?
    - 당연하지만 변하지 않는 것을 상수로 선언하면 된다. 추가적으로, 상수를 변경하면 오류가 발생하기 때문에 일단 모두 상수로 선언하고 오류가 발생하는 부분만 변수로 바꾸면 된다.

## 2.2 자료형 변환

자바스크립트는 자료형을 미리 결정하지 않고 코드를 실행하는 동안 결정하는 동적 타입 언어다.

### 2.2.1 자료형 체크

- **강력한 자료형 체크** : 변수를 선언할 때 자료형을 미리 결정 하는 것
- **느슨한 자료형 체크** : 미리 변수의 자료형을 지정하지 않고 변수에 값을 할당하면 자료형이 결정됨

### 2.2.2 자동형 변환

- 문자열 값을 사칙 연산에 사용하면 자동으로 **문자열→숫자**로 변환된다.
- 숫자와 문자를 더하면 `+` 를 연결연산자로 인식해 **숫자 → 문자열**로 변환된다.
- 숫자와 문자를 빼면 **문자열 → 숫자**로 변환된다.

### 2.2.3 자료형 변환 함수

| 함수 | 설명 |
| --- | --- |
| Number() | 문자, 논리 → 숫자 |
| parseInt() | 문자열 → 정수 |
| parseFloat | 문자열 → 실수 |
| string() | 숫자, 논리 → 문자열 |
| Boolean() | 괄호값 → 논리 |

# 3. 연산자

---

## 3.1 ==, !=, ===, !==

- `==`, `!=` 연산자는 피연산자의 자료형을 자동으로 변환해 비교한다.

```jsx
3 == "3" //true
3 != "3" //false
```

- `===`, `!==` 연산자는 피연산자의 자료형을 변환하지 않는다.

```jsx
3 === "3" //fasle
3 != "3" //true
```

## 4.2 짧은 조건문

```jsx
<불 표현식> || <불표현식이 거짓말일 때 실행할 문장>
<불 표현식> && <불 표현식이 참일 때 실행할 문장> 

// 설명
true || OOO : 좌변이 참이기 때문에 어떤 우변이 들어와도 참
false || OOO : 좌변이 거짓이기 때문에 우변이 참인지 검사 수행
```

```jsx
// 예제
<script>
	var input = Mumber(prompt('숫자를 입력해쥇요.', '숫자'))
	input % 2 == 0 || alert('홀수입니다.')
	input % 2 == 0 && alert('짝수입니다.')
</script>
```
